# CSS 선택자와 쉼표의 역할

## 쉼표의 역할 차이

### 1️⃣ `.front, .back` (쉼표 O) - 현재 상태

```css
.front, .back {
    border: 3px dotted rgb(192, 192, 224);
    position: absolute;
    /* ... 기타 스타일 ... */
}
```

**의미**: 여러 선택자(Multiple Selectors) → 같은 스타일을 **별도의** 두 요소에 적용
- `.front` 요소와 `.back` 요소가 각각 독립적으로 이 스타일을 받음
- CSS에서는 이렇게 해석됨:
  ```css
  .front { /* 스타일 */ }
  .back { /* 스타일 */ }
  ```

---

### 2️⃣ `.front.back` (쉼표 X인 경우) - 만약 쉼표가 없다면?

```css
.front.back {
    border: 3px dotted rgb(192, 192, 224);
    /* ... */
}
```

**의미**: 복합 선택자(Compound Selector) → **동시에 두 클래스를 가진** 하나의 요소만 선택
- HTML에서 `<div class="front back">` 이런 식으로 두 클래스를 모두 가진 경우에만 적용
- 현재 코드에서는 `.front`와 `.back`이 별개 요소이므로 **스타일이 전혀 적용되지 않음**
- **→ 이것이 구조가 망가지는 이유!**

---

### 3️⃣ `.card:hover .inner` (쉼표 없음) - 왜 쉼표가 없을까?

```css
.card:hover .inner {
    transform: rotateY(180deg);
}
```

**의미**: 하위 선택자(Descendant Combinator) 구조
- "`.card`에 hover 상태일 때, **그 내부의** `.inner` 요소"를 의미
- 공백(` `)이 선택자 연결 기호 역할을 함

**만약 쉼표를 붙인다면?**
```css
.card:hover, .inner {
    /* 잘못된 문법! */
}
```
→ ".card:hover 상태" **또는** ".inner"를 선택한다는 뜻이 되어 완전히 다른 의미

---

## 정리표

| 선택자 | 역할 | 쉼표 사용 | 예시 |
|-------|------|---------|------|
| `.front, .back` | 두 개의 **별개 요소** | ✅ **필수** | 여러 요소에 같은 스타일 |
| `.front.back` | **동시에** 두 클래스 보유 | ❌ 안 함 | 두 클래스 동시 선택 |
| `.card:hover .inner` | **포함 관계** (부모→자식) | ❌ 안 함 | 특정 상태의 자식 선택 |

---

## 핵심 포인트

✨ **쉼표 있음(`A, B`)**: "A **또는** B" → 두 요소를 독립적으로 선택
✨ **쉼표 없음(`AB`)**: "A **그리고** B" → 두 조건을 동시에 만족하는 요소만 선택
✨ **공백(`A B`)**: "A의 자식 B" → 포함 관계 표현
