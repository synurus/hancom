MDN @media 페이지 요약
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------
**@media**는 CSS의 @규칙(at-rule) 중 하나로, 스타일 시트의 일부를 특정 미디어 쿼리 결과에 따라 조건부로 적용할 때 사용합니다. 즉, 화면 크기, 인쇄 여부 같은 조건을 만족하는 장치에서만 CSS를 적용할 수 있어요.

**기본 사용법:**
```css
@media screen and (min-width: 900px) {
  article {
    padding: 1rem 3rem;
  }
}
```

**주요 용도:**
- 반응형 웹 디자인 (화면 크기별 스타일 다르게 적용)
- 인쇄용 스타일 따로 지정 (`@media print`)
- 다크모드, 애니메이션 감소 등 사용자 설정 감지

**최신 문법 (Media Queries Level 4):**
```css
@media (height > 600px) {
  body { line-height: 1.4; }
}
```
비교 연산자(`>`, `<=` 등)를 써서 더 간결하게 작성 가능해요.

MDN :root 페이지 요약
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------
**`:root`** 의사 클래스는 문서 트리의 루트 요소를 선택하는 CSS 선택자예요. HTML에서는 `<html>` 요소가 루트이므로, `:root`는 명시도(specificity)가 더 높다는 점만 빼면 `html` 선택자와 동일하게 동작합니다.

**주요 용도: 전역 CSS 변수 선언**

가장 많이 쓰이는 곳은 전역 커스텀 프로퍼티(CSS 변수)를 선언할 때예요.

```css
:root {
  --main-color: hotpink;
  --pane-padding: 5px 42px;
}
```

이렇게 선언하면 문서 전체 어디서든 `var(--main-color)`처럼 불러와서 재사용할 수 있어요.

**정리**
- `:root` = `<html>` 선택 (명시도만 더 높음)
- 주로 CSS 변수를 전역으로 선언할 때 사용
- 브라우저 호환성: 2015년 7월부터 널리 지원되는 안정적인 기능

MDN box-shadow 요약
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------
**`box-shadow`**은 요소의 테두리 주변에 그림자 효과를 넣는 CSS 속성이에요. 쉼표로 구분해서 여러 개의 그림자를 겹쳐 적용할 수도 있습니다.

**문법:**
```css
box-shadow: <x-offset> <y-offset> <blur-radius> <spread-radius> <color>;
```

**값 설명:**
- **offset-x, offset-y** (필수): 그림자의 가로/세로 위치
- **blur-radius** (선택): 흐림 정도, 클수록 그림자가 크고 흐릿해짐
- **spread-radius** (선택): 그림자 크기 확장/축소
- **color** (선택): 그림자 색상, 생략 시 부모의 `color` 값 사용
- **inset** (선택): 넣으면 그림자가 요소 안쪽으로 들어감 (기본은 바깥쪽)

**예시:**
```css
box-shadow: 10px 5px 5px red;              /* 기본 그림자 */
box-shadow: inset 5em 1em gold;            /* 안쪽 그림자 */
box-shadow: 3px 3px red, -1em 0 0.4em olive; /* 그림자 2개 */
```

**참고:**
- `border-radius`가 설정된 요소는 그림자도 같이 둥글게 나와요
- 여러 그림자를 쓰면 먼저 쓴 것이 위에 표시됨
- 2015년 7월부터 널리 지원되는 안정적인 기능이에요

핵심 요약
• ::before — 콘텐츠 앞에 장식
• ::after — 콘텐츠 뒤에 장식
• content 속성 필수 — 없으면 안 나타남
• HTML 수정 없이 순수 CSS로 별·따옴표·밑줄

왜 사용할까? — 장식용 빈 <span>을 HTML에 직접 넣지 않고 CSS 한 곳에서만 처리
• HTML이 깔끔하게 유지됨 (장식 태그가 늘어나지 않음)

• 별·따옴표는 "내용이 아님" → 스크린리더가 읽지 않음 (접근성 ↑)

MDN display 페이지 요약
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------
**`display`**은 요소를 블록 박스로 취급할지 인라인 박스로 취급할지, 그리고 그 자식 요소들에 어떤 레이아웃 방식(flow, grid, flex 등)을 사용할지 결정하는 CSS 속성이에요.

**핵심 개념: outer + inner 타입**
- **outer(외부)**: `block` / `inline` — 요소가 주변 흐름에서 어떻게 배치되는지
- **inner(내부)**: `flow` / `grid` / `flex` / `table` 등 — 자식들을 어떤 방식으로 배치하는지

**주요 값:**
```css
display: block;         /* 블록 요소 (줄바꿈 O) */
display: inline;        /* 인라인 요소 (줄바꿈 X) */
display: inline-block;  /* 인라인처럼 배치되지만 블록처럼 크기 조절 가능 */
display: flex;          /* 플렉스 컨테이너 */
display: grid;          /* 그리드 컨테이너 */
display: none;          /* 요소를 아예 렌더링하지 않음 */
display: contents;      /* 요소 자체는 사라지고 자식만 남음 */
```

**최신 문법(멀티 키워드)**도 지원해요:
```css
display: inline flex;   /* = inline-flex */
display: block grid;    /* = grid */
```

**주의할 점:**
- `display: none`은 접근성 트리에서도 완전히 제거되어 스크린리더가 읽지 않음 (시각적으로만 숨기려면 `visibility` 사용 권장)
- 2015년 7월부터 널리 지원되는 안정적인 기능
