# JSX 안에서 변수 선언 시 발생하는 오류

## 문제 상황

```jsx
const App = () => {
  return (
    <>
    const list = ['react', 'props', 'map']
    <Tag tags={list} />
    </>
  )
}
```

위 코드를 실행하면 `<Tag tags={list} />` 부분에서 다음 오류가 발생한다.

```
'list' is not defined
```

## 원인

JSX의 `<>...</>` (또는 임의의 태그) 안에 들어있는 모든 내용은 **JSX 자식(children)**으로 취급된다.

- `{}`로 감싸지 않은 텍스트/코드는 **그냥 문자열(텍스트 노드)**로 파싱된다. 실제 JS 코드로 실행되지 않는다.
- 따라서 `const list = ['react', 'props', 'map']`는 변수를 선언하는 게 아니라, 그 글자 그대로를 화면에 표시하려는 텍스트로 처리된다.
- 그 다음 줄의 `{list}`는 `{}`로 감쌌기 때문에 **진짜 JS 표현식**으로 해석되는데, 이때 `list`라는 변수가 실제로 선언된 적이 없으므로 `'list' is not defined` 오류가 발생한다.

## 핵심 규칙

> JSX 안에는 **표현식(expression)**만 `{}`로 감싸서 넣을 수 있다.
> `const`, `let`, `if`, `for` 같은 **문장(statement)**은 JSX 안에 넣을 수 없다.

| 종류 | JSX 안에서 사용 가능? | 예시 |
|---|---|---|
| 표현식 (expression) | ✅ 가능 (`{}`로 감싸서) | `{list}`, `{a + b}`, `{list.map(...)}` |
| 문장 (statement) | ❌ 불가능 | `const list = [...]`, `if (...) {...}`, `for (...) {...}` |

## 해결 방법

변수 선언은 **JSX 바깥, `return` 이전**에서 해야 한다.

```jsx
const App = () => {
  const list = ['react', 'props', 'map']   // return 밖에서 선언 (문장이므로 OK)

  return (
    <>
    <Tag tags={list} />   {/* {} 안에서는 표현식이므로 OK */}
    </>
  )
}
```

이렇게 하면 `list`는 컴포넌트 함수 스코프에 정상적으로 선언되고, JSX 안에서는 `{list}`로 그 값을 참조(표현식)만 하게 되어 정상 작동한다.

# React에서 이미지를 불러오는 2가지 방식

## 1️⃣ 문자열 경로 방식 (안 됨)
```jsx
<img src="./assets/image_top.jpg" alt="menu-top" />
```

### 동작 원리
- **런타임**에 경로를 해석
- 브라우저가 직접 파일을 찾으려고 시도
- 상대 경로를 계산할 때 기준점이 `public` 폴더 또는 현재 웹 주소

### 문제점
❌ 경로가 복잡할 때 오류 발생
❌ 파일 위치가 바뀌면 런타임에만 오류 감지
❌ 파일이 번들에 포함되지 않을 수 있음
❌ 경로 오류가 있어도 빌드 시점에는 발견 안 됨

---

## 2️⃣ Import 방식 (작동함) ✅
```jsx
import imageTop from './assets/image_top.jpg'

const Menu = () => {
    return (
        <img src={imageTop} alt="menu-top" />
    )
}
```

### 동작 원리
- **빌드 타임**에 경로를 해석
- webpack 같은 번들러가 이미지 파일을 처리
- 이미지가 자동으로 프로젝트 번들에 포함됨
- 최적화 처리 (압축, 해시 이름 생성 등)

### 장점
✅ 파일이 없으면 빌드 시점에 오류 발생 (미리 감지)
✅ 경로 오류를 사전에 방지
✅ 파일을 확실하게 번들에 포함
✅ 성능 최적화 자동 처리
✅ TypeScript 지원 가능
✅ 파일 이름이 바뀌어도 에디터에서 감지

---

## 비교표

| 항목 | 문자열 경로 | Import 방식 |
|---|---|---|
| 처리 시점 | 런타임 | 빌드 타임 |
| 오류 감지 | 런타임에만 | 빌드 시점에 |
| 파일 포함 | 불확실 | 확실함 |
| 성능 최적화 | 없음 | 자동 |
| 권장도 | ❌ | ✅ |

---

## 실제 코드 비교

### ❌ 문자열 경로 방식 (사용 금지)
```jsx
const Menu = () => {
    return (
        <img src="./assets/image_top.jpg" alt="menu-top" />
    )
}
```

### ✅ Import 방식 (권장)
```jsx
import imageTop from './assets/image_top.jpg'

const Menu = () => {
    return (
        <img src={imageTop} alt="menu-top" />
    )
}
```

---

## 언제 어느 방식을 쓸까?

### Import 방식을 써야 할 때
- React 프로젝트에서 컴포넌트 내 이미지
- 크기가 작은 이미지들
- 정적 이미지 (코드 작성 시점에 경로가 고정됨)

### 문자열 경로를 써야 할 때
- `public` 폴더의 이미지만 가능
- 사용자가 업로드한 이미지 (동적 경로)
- 외부 URL의 이미지

---

## 핵심 정리

> **React에서는 import 방식이 정상입니다.**
> 
> - Import는 webpack이 처리 → 확실하고 안전
> - 문자열 경로는 런타임에만 처리 → 불안정하고 오류 발생 가능
> - 정적 이미지는 import, 동적 이미지만 문자열 경로 사용
